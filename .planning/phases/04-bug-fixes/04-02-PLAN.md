  <action>
Fix the duplicate padding-bottom classes on the fixed bottom bar (per D-06).

Root cause: Line 90 has both `pb-[env(safe-area-inset-bottom)]` and `pb-4`. In Tailwind CSS v4, when two utilities target the same CSS property, the last one wins. So `pb-4` (1rem) overrides the safe-area-inset value, causing notched devices (iPhone with home indicator) to lose their safe area padding.

Change the className on line 90 from:
```
"fixed bottom-0 left-0 right-0 pb-[env(safe-area-inset-bottom)] bg-black/10 backdrop-blur-sm px-4 pt-4 pb-4"
```

To:
```
"fixed bottom-0 left-0 right-0 pb-[max(1rem,env(safe-area-inset-bottom))] bg-black/10 backdrop-blur-sm px-4 pt-4"
```

Changes:
1. Remove the first `pb-[env(safe-area-inset-bottom)]` class
2. Remove the trailing `pb-4` class
3. Add a single `pb-[max(1rem,env(safe-area-inset-bottom))]` class that uses CSS `max()` to apply whichever is larger: 1rem (equivalent to pb-4) or the device's safe area inset
4. Keep `pt-4` unchanged (top padding is not conflicting)

This ensures:
- Non-notched devices: 1rem bottom padding (same as before)
- Notched devices: safe-area-inset-bottom padding (which is larger than 1rem on iPhone)
  </action>
