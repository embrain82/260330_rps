Also fix the animation final value in the useEffect:

Change Phase 3 (final reveal snap) from:
```typescript
await animate(
  scope.current,
  { rotateY: 0, scale: [0.8, 1.1, 1] },
  { duration: 0.3, ease: 'easeOut' }
)
```

To:
```typescript
await animate(
  scope.current,
  { rotateY: 1080, scale: [0.8, 1.1, 1] },
  { duration: 0.3, ease: 'easeOut' }
)
```

Why: The animation goes 0->720->900. To show the FRONT face at the end, the final value must be a multiple of 360. Going from 900 to 1080 (= 3 full rotations) is a continuous forward motion that lands on the front face. The previous `rotateY: 0` was a jarring reverse rotation from 900 to 0.

4. Back face: #FF6B6B bg, 3 small icons (w-6 h-6), `backfaceVisibility: 'hidden'`, `transform: 'rotateY(180deg)'`, `data-testid="card-back"`
5. Phase 3 animation: `rotateY: 0` -> `rotateY: 1080`
6. Remove `bg-white` from the container div (color is now on individual faces)
7. Keep all imports unchanged (RockIcon, PaperIcon, ScissorsIcon already imported)
