    **Step 3: Create `src/store/gameStore.ts`** implementing the Zustand store per RESEARCH.md Pattern 1.

    The store interface:
    ```typescript
    interface GameStore {
      // State (mirrors GameState from types.ts)
      phase: Phase
      round: number            // 0-indexed
      drawCount: number
      playerChoice: Choice | null
      aiChoice: Choice | null
      lastOutcome: Outcome | null
      roundResults: Array<{ outcome: Outcome; round: number }>
      session: SessionPayload

      // Actions
      start: () => void
      select: (choice: Choice) => void
      revealDone: () => void
      advance: () => void
      retry: () => void
    }
    ```

    Implementation details:
    - `start()`: Creates new session via `createSession()`, resets round/drawCount/choices/results, sets phase='selecting'
    - `select(choice)`: Calls `pickAiChoice(round, choice, drawCount)` to get AI choice, sets both choices atomically with phase='revealing' in ONE `set()` call (Pitfall 1 from RESEARCH.md -- atomic update prevents null reads)
    - `revealDone()`: Calls `determineOutcome(playerChoice!, aiChoice!)`, sets lastOutcome, sets phase='result'
    - `advance()`: Branches on lastOutcome:
      - 'draw': phase='selecting', drawCount+1, clear choices/outcome
      - 'lose': phase='gameover', append to roundResults, finalize session
      - 'win' + nextRound >= TOTAL_ROUNDS: phase='victory', append to roundResults, finalize session
      - 'win' + nextRound < TOTAL_ROUNDS: phase='selecting', round+1, drawCount=0, clear choices/outcome, append to roundResults
    - `retry()`: Full reset to initial state with new session

    Define `initialState` as a const object for test resets. Export it alongside `useGameStore`.

    Import from `'@/lib/rps'`: `pickAiChoice`, `determineOutcome`, `createSession`, `finalizeSession`, `TOTAL_ROUNDS`, and types `Choice`, `Phase`, `Outcome`, `SessionPayload`.
