# Phase 2 — UI Design Contract

> Visual and interaction contract for the fully playable RPS Challenge game UI. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (custom game components -- arcade-style UI is entirely bespoke) |
| Icon library | Custom inline SVG components (Rock.tsx, Paper.tsx, Scissors.tsx) -- cartoon glove style per D-01 |
| Font | Geist (sans) via `next/font/google` -- already installed in layout.tsx |

**Rationale for no shadcn:** This is an arcade-style mini-game with custom SVG illustrations, custom animations, and bespoke visual identity (D-01, D-02). shadcn/Radix primitives target form-heavy UIs and would add 30KB+ overhead with no visual payoff. All interactive elements (3 choice buttons, 1 start button, 1 retry button) are trivially accessible without a component library.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between step dots (D-06) |
| sm | 8px | Compact element spacing, gap between label and icon in choice buttons |
| md | 16px | Default element spacing, padding inside cards, gap between battle area elements |
| lg | 24px | Section padding, gap between round indicator and battle area |
| xl | 32px | Layout gaps, padding around choice button bar (D-05) |
| 2xl | 48px | Major section breaks, vertical spacing between idle screen elements (D-04) |
| 3xl | 64px | Page-level spacing, top/bottom safe area padding |

Exceptions:
- **44px minimum touch target** for all interactive elements (choice buttons, start button, retry button) per UI-01 and WCAG 2.5.5. Applied as `min-h-[44px] min-w-[44px]` on the outer `<button>` element, not inner content.
- **80px choice button height** for choice buttons (D-05) -- large enough for SVG hand icon + label within bottom bar.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | Rule description on idle screen (D-04), round indicator labels |
| Label | 14px | 600 (semibold) | 1.4 | Choice button labels ("가위", "바위", "보"), step dot captions |
| Heading | 24px | 700 (bold) | 1.2 | Round result text ("승리!" / "패배..." / "무승부! 다시!") per D-12, D-13 |
| Display | 36px | 800 (extrabold) | 1.1 | Victory/gameover screen headline ("축하합니다!" / "아쉽네요...") per D-07 |

**Font:** Geist Sans (`--font-geist-sans`). All text uses this single font family. Geist Mono is available but not used in game UI.

**Note on weights:** Geist is a variable font supporting 100-900. The game's arcade aesthetic (D-02) demands bolder weights than typical apps, so 4 weights are used: 400, 600, 700, 800. This is an exception to the standard 2-weight guideline, justified by the game's visual identity requiring clear weight hierarchy across body/label/heading/display roles.

## Color

### Base Palette

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F8F9FA` (near-white) | Page background behind game container, idle screen surface |
| Secondary (30%) | `#FFFFFF` | Card surfaces, battle area background, choice button background |
| Accent (10%) | `#FF6B6B` (coral red) | Primary CTA button (start/retry), active choice button ring, current step dot |
| Destructive | `#DC3545` | Game-over flash background, defeat result text |

Accent reserved for: start game button fill, retry button fill, current round step dot, active/selected choice button border highlight. Never used for decorative or passive elements.

Background transitions per round to visualize escalating tension:

| Round | CSS Variable | Value | Description |
|-------|-------------|-------|-------------|
| Round 1 (index 0) | `--bg-round-0` | `#4A90D9` | Calm blue |
| Round 2 (index 1) | `--bg-round-1` | `#2CA5A5` | Teal |
| Round 3 (index 2) | `--bg-round-2` | `#E8B84A` | Warm yellow |
| Round 4 (index 3) | `--bg-round-3` | `#E87E3A` | Orange |
| Round 5 (index 4) | `--bg-round-4` | `#D94040` | Intense red |

Applied as CSS custom properties in `:root`. Background transitions with motion `animate` over 0.5s on round change. Text on round backgrounds uses white (`#FFFFFF`) for contrast.

## Animation Contract

### Screen Transitions (motion AnimatePresence, mode="wait")

| Transition | Enter | Exit | Duration |
|------------|-------|------|----------|
| idle -> selecting | `opacity: 0 -> 1` | `opacity: 1 -> 0` | 0.3s |
| selecting/revealing/result (grouped as "play") | `opacity: 0 -> 1` | `opacity: 1 -> 0, scale: 1 -> 0.95` | 0.3s |
| play -> victory/gameover | `opacity: 0, scale: 0.9 -> opacity: 1, scale: 1` | `opacity: 1 -> 0` | 0.5s spring |

### Suspense Card Shuffle (D-08, FX-03)

| Phase | Animation | Duration | Easing |
|-------|-----------|----------|--------|
| Fast shuffle | `rotateY: 0 -> 720` | 0.6s | linear |
| Deceleration | `rotateY: 720 -> 900` | 0.6s | easeOut |
| Final reveal snap | `rotateY -> 0, scale: 0.8 -> 1.1 -> 1` | 0.3s | easeOut |
| **Total** | | **~1.5s** | |

Implemented with motion `useAnimate` imperative sequence. Fires when phase = `'revealing'`. Calls `revealDone()` on completion.

### Choice Reveal (FX-04)

| Animation | Property | Duration | Easing |
|-----------|----------|----------|--------|
| Card flip | `rotateY: 180 -> 0` | 0.3s | spring (stiffness: 200, damping: 20) |
| Scale pop | `scale: 0.8 -> 1` | 0.3s | spring |

Player card and AI card reveal simultaneously after suspense completes.

### Victory Confetti (D-09, FX-01)

| Burst | Origin | Delay | Particle Count |
|-------|--------|-------|----------------|
| Center | `{ x: 0.5, y: 0.7 }` | 0ms | 50 |
| Left | `{ x: 0.1, y: 0.8 }` | 400ms | 50 |
| Right | `{ x: 0.9, y: 0.8 }` | 800ms | 50 |
| Center wave 2 | `{ x: 0.5, y: 0.5 }` | 1200ms | 80 |

Options: `spread: 80, startVelocity: 45, gravity: 1.2, ticks: 250, disableForReducedMotion: true, useWorker: true`. Total duration: 3-4s. Cleanup via `confetti.reset()` in useEffect return.

### Defeat Effect (D-10, FX-02)

| Step | Animation | Duration | Easing |
|------|-----------|----------|--------|
| Shake | `x: [0, -4, 4, -4, 4, -2, 2, 0]` | 0.3s | easeOut |
| Desaturate | `filter: grayscale(0%) -> grayscale(100%)` | 0.5s | easeIn |
| **Total** | | **~0.8s** | |

Implemented with motion `useAnimate` sequential await. Crack overlay effect is at Claude's discretion per CONTEXT.md.

### Result Text Flash (D-12, D-13)

| Result | Background flash | Text scale | Auto-advance delay |
|--------|-----------------|------------|-------------------|
| Win | green `#22C55E` -> round bg | `scale: 0 -> 1.2 -> 1` | 1.0s then auto-advance |
| Lose | red `#EF4444` -> hold | `scale: 0 -> 1.2 -> 1` | 1.0s then transition to gameover |
| Draw | yellow `#EAB308` -> round bg | `scale: 0 -> 1.1 -> 1` | 0.8s then auto-advance to same round |

### Round Flow Timing (D-11)

| Step | Duration | Auto |
|------|----------|------|
| Player taps choice | instant | no |
| Suspense shuffle | ~1.5s | yes |
| Choice reveal | 0.3s | yes |
| Result display | 1.0s (win/lose) / 0.8s (draw) | yes |
| Transition to next | 0.3s | yes |
| **Total per round** | **~3.1s** (win) / **~2.9s** (draw) | |

No "next" button -- all transitions are automatic per D-11.

### Idle Ambient Animations (CSS @keyframes)

| Element | Animation | Duration | Loop |
|---------|-----------|----------|------|
| Start button | Gentle pulse `scale: 1 -> 1.05 -> 1` | 2s | infinite |
| Choice button hover/press | `scale: 1 -> 0.95` on press, `1 -> 1.05` on hover | 0.15s | none |
| Step dots (current) | Soft glow `opacity: 0.7 -> 1 -> 0.7` | 1.5s | infinite |

Implemented as CSS `@keyframes` in globals.css `@theme` block. Ambient animations use CSS, not motion, per RESEARCH.md guidance.

## Layout Contract

### Mobile-First Responsive (UI-01)

| Breakpoint | Viewport | Layout Adjustments |
|------------|----------|--------------------|
| Base (mobile) | 360px - 767px | Full-width, bottom bar choice buttons, stacked battle area |
| Tablet+ | 768px+ | Centered container max-width 480px, same layout proportionally scaled |

The game is designed for mobile-first (360px minimum). On larger screens, it centers in a max-480px container to maintain mobile proportions -- this is an embedded mini-game, not a desktop app.

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Touch targets | All buttons min 44x44px (UI-01) |
| Reduced motion | `disableForReducedMotion: true` on confetti. motion respects `prefers-reduced-motion` via `useReducedMotion()` hook -- skip non-essential animations, keep state transitions functional |
| Focus management | After screen transition, focus moves to primary action (start button on idle, first choice button on play, retry button on end) |
| Button semantics | All interactive elements are `<button>` elements with descriptive text content |
| Color contrast | All text on colored backgrounds meets WCAG AA (4.5:1). White text on round backgrounds verified: blue #4A90D9 = 4.6:1, teal #2CA5A5 = 3.2:1 (use darker teal #1E8A8A for text areas), yellow #E8B84A = 2.1:1 (use dark text #1A1A2E on yellow), orange #E87E3A = 3.0:1 (use dark text #1A1A2E), red #D94040 = 4.8:1 |
| Language | `lang="ko"` on html element for Korean content |

**Contrast adjustments for round backgrounds:**
- Rounds 1, 5 (blue, red): White text is AA compliant
- Rounds 2, 3, 4 (teal, yellow, orange): Use dark text `#1A1A2E` for body copy. White text acceptable for heading-size (24px+) text at AA Large (3:1)
